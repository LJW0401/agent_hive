# Agent Hive — 功能规划文档

> 终端智能体并行管理 — 基于 Web 的多终端管理器

## 一、产品定位

通过 Web 界面同时管理多个终端会话，用于并行运行 AI Agent 或其他终端任务。支持本地部署、局域网访问和内网穿透。

---

## 二、核心功能

### 2.1 项目容器

页面的基本单元是**项目容器**，每个容器包含：

- **容器名称**：显示在容器顶部，可随时修改，用于标识当前项目/任务
- **关闭按钮**：容器顶部提供关闭按钮，可直接销毁整个容器（含终端）
- **左侧 — 待办事项**：管理该项目相关的待办清单
- **右侧 — 终端**：一个完整的终端实例

#### 容器生命周期

- **创建**：点击空闲区块的"新建项目"按钮创建
- **销毁方式一**：用户在终端内输入 `exit` 退出
- **销毁方式二**：点击容器顶部的关闭按钮
- 容器不会因页面关闭或断连而终止

### 2.2 终端能力

- **完整终端模拟**：支持 ANSI 颜色、光标定位、TUI 程序（vim、htop、tmux 等）
- **全量回滚缓冲**：保存终端的全部输出历史，用户可任意回滚查看

### 2.3 待办事项

- 每个项目容器内置独立的待办清单
- 支持新建、删除、更新内容、标记完成/未完成
- 支持上下拖动排序
- 待办数据随容器持久化存储

### 2.4 Web 界面布局

- **网格布局**：默认每页 2×2（4 个项目容器）
- **响应式自适应**：根据浏览器窗口大小自动调整排列
- **空闲区块**：未被容器占用的格子显示"新建项目"占位按钮
- **多页管理**：支持多页，通过左右切换浏览不同页面
- **拖拽排序**：容器可在页面内拖拽调整位置
- **跨页移动**：容器可拖拽移动到其他页面

### 2.5 会话持久化

#### 浏览器重连（后端未重启）

- **后端常驻运行**：终端进程由后端服务维持，生命周期独立于浏览器连接
- **完整恢复**：关闭网页后重新打开，恢复全部容器（终端实例、布局位置、容器名称、待办事项、终端输出历史）

#### 机器重启（后端重启）

- **必须恢复**：页面布局、容器名称、待办事项（持久化存储到磁盘）
- **终端进程**：重启后终端进程已丢失，容器内终端显示为已断开状态，用户可选择重新打开终端
- **终端历史**：尽力恢复（如有落盘的输出记录则加载），但不保证完整

### 2.6 访问与会话控制

- **本地部署**：服务运行在本机，监听局域网可达的地址
- **多种访问方式**：支持 localhost 直连、局域网 IP 访问、内网穿透（frp/cloudflared 等）访问
- **单用户独占模式**：同一时间只允许一个设备进行操作
- **设备抢占机制**：新设备登录后获得控制权，旧设备降级为只读模式（可查看终端输出，不可输入）
- **认证方式**：配置文件中设定密码/token，同时支持配置机器唯一标识码白名单

## 三、技术方案

### 3.1 后端：Go

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Go | 编译为单二进制文件，无运行时依赖，部署即复制 |
| PTY | creack/pty | Go 生态最成熟的 PTY 库 |
| WebSocket | gorilla/websocket | 稳定，并发性能好 |
| HTTP | Gin 或 net/http | 提供 API + 内嵌前端静态资源 |
| 持久化 | SQLite (go-sqlite3) | 单文件数据库，存布局和待办 |
| 静态资源 | go:embed | 前端打包产物内嵌到二进制中 |

**选 Go 的理由：**

- 长期驻留的后台服务，内存占用低，稳定性好
- 编译产物为单个可执行文件，不需要用户安装 Node/npm 环境
- 前端资源通过 `go:embed` 打包，最终交付仅一个文件
- 原生协程管理多个 PTY 会话天然适合

### 3.2 前端：React + TypeScript

| 组件 | 选择 | 理由 |
|------|------|------|
| 框架 | React + TypeScript | 组件化清晰，类型安全 |
| 终端 | xterm.js + xterm-addon-fit | Web 终端事实标准，VS Code 同款 |
| 拖拽 | dnd-kit | React 生态最好的拖拽库，支持网格 + 跨容器 |
| 组件库 | shadcn/ui (Radix) | 现代风格，组件源码直接可改，定制性强 |
| 图标 | Lucide Icons | 与 shadcn/ui 搭配，风格统一 |
| 动画 | Framer Motion | 拖拽过渡、页面切换动画流畅 |
| 样式 | Tailwind CSS | 快速布局，响应式自适应开箱即用 |
| 主题 | 深色主题 | 终端场景天然适合，视觉风格统一 |
| 通信 | WebSocket + REST | 终端 I/O 走 WebSocket，布局/待办走 REST |

**视觉方向：** 深色底 + 微妙边框 + 柔和阴影 + 流畅过渡动画（类似 Linear / Vercel Dashboard 风格）

### 3.3 数据存储策略

| 数据类型 | 存储方式 | 理由 |
|------|------|------|
| 布局、容器名称、待办事项 | SQLite | 结构化、事务安全、机器重启后必须恢复 |
| 终端输出历史 | 磁盘文件（按容器 ID 追加写入） | 数据量大，流式写入性能好，重启后尽力恢复 |

### 3.4 交付形态

```
./agent-hive          # 单个二进制文件，内嵌前端资源
./config.yaml         # 配置文件（端口、密码/token、机器白名单）
./data/               # 运行时数据目录
  ├── agent_hive.db   # SQLite 数据库
  └── terminals/      # 终端输出历史文件
```

启动方式：`./agent-hive`，打开浏览器即用。
