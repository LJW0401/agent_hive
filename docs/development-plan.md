# Agent Hive — 开发计划

> 基于 [功能规划文档](./feature-plan.md) 拆分的分阶段开发计划

---

## 第一阶段：项目骨架与单终端跑通

**目标：** 最小可运行原型——浏览器打开能看到一个终端并正常交互

### 后端

- [x] 初始化 Go 项目，搭建目录结构
- [x] 集成 creack/pty，实现单个 PTY 会话的创建与管理
- [x] 集成 gorilla/websocket，实现终端 I/O 的 WebSocket 通道
- [x] HTTP 服务，提供静态资源托管

### 前端

- [x] 初始化 React + TypeScript + Vite 项目
- [x] 集成 Tailwind CSS + shadcn/ui，搭建深色主题基础样式
- [x] 集成 xterm.js，实现单个终端组件
- [x] WebSocket 连接后端，终端可正常输入输出

### 验收标准

- `./agent-hive` 启动后，浏览器访问能看到一个可交互的终端
- 支持颜色、vim 等 TUI 程序

---

## 第二阶段：项目容器与网格布局

**目标：** 多个项目容器在页面上以网格形式展示

### 后端

- [ ] 多 PTY 会话管理（创建、销毁、列表查询）
- [ ] 每个终端独立的 WebSocket 通道
- [ ] REST API：创建容器、删除容器、获取容器列表

### 前端

- [ ] 项目容器组件（顶栏名称 + 关闭按钮 + 左侧待办区 + 右侧终端区）
- [ ] 网格布局实现（默认 2×2，响应式自适应）
- [ ] 空闲区块显示"新建项目"按钮
- [ ] 容器名称的编辑功能

### 验收标准

- 可以创建多个容器，网格排列展示
- 每个容器有独立可交互的终端
- 窗口缩放时布局自适应

---

## 第三阶段：待办事项

**目标：** 每个项目容器左侧的待办清单功能完整可用

### 后端

- [ ] 集成 SQLite，设计待办事项表结构
- [ ] REST API：待办的增删改查、标记完成、排序更新

### 前端

- [ ] 待办事项列表组件
- [ ] 新建、编辑（行内编辑）、删除、标记完成/未完成
- [ ] 集成 dnd-kit 实现上下拖动排序
- [ ] 待办区与终端区的布局比例协调

### 验收标准

- 待办清单的增删改查、完成标记、拖动排序全部可用
- 刷新页面后待办数据不丢失

---

## 第四阶段：多页管理与拖拽

**目标：** 支持多页切换，容器可拖拽排序和跨页移动

### 后端

- [ ] SQLite 存储布局数据（容器所在页、位置顺序）
- [ ] REST API：布局更新（页面、排序）

### 前端

- [ ] 多页切换 UI（页面指示器、左右切换按钮/手势）
- [ ] 页面切换动画（Framer Motion）
- [ ] 容器拖拽排序（dnd-kit 网格拖拽）
- [ ] 跨页拖拽移动（拖到边缘或页面指示器触发换页）

### 验收标准

- 容器超过 4 个时自动分页
- 左右切换页面流畅
- 容器可拖拽排序、跨页移动，松手后位置持久化

---

## 第五阶段：会话持久化

**目标：** 关闭浏览器可恢复，机器重启后布局和待办可恢复

### 后端

- [ ] 终端输出实时追加写入磁盘文件（按容器 ID 存储）
- [ ] 浏览器重连时推送终端输出历史
- [ ] 后端启动时从 SQLite 恢复容器元数据（名称、布局、待办）
- [ ] 后端启动时检测已断开的终端，标记为"已断开"状态
- [ ] 容器重新打开终端的 API

### 前端

- [ ] 重连逻辑：WebSocket 断开后自动重连，恢复终端显示
- [ ] 终端"已断开"状态的 UI 展示（提示 + 重新打开按钮）
- [ ] 连接状态指示器

### 验收标准

- 关闭浏览器重新打开，所有容器、终端输出、待办事项完整恢复
- 重启服务后，布局和待办恢复，终端显示为已断开并可重新打开

---

## 第六阶段：访问控制与认证

**目标：** 单用户认证、设备独占、只读降级

### 后端

- [ ] config.yaml 配置文件加载（端口、密码/token、机器白名单）
- [ ] 登录认证 API（token/密码校验）
- [ ] 机器标识码白名单校验
- [ ] 会话管理：追踪当前活跃设备
- [ ] 设备抢占逻辑：新登录踢出旧设备，通过 WebSocket 通知旧设备

### 前端

- [ ] 登录页面
- [ ] 被抢占时的只读模式 UI（输入禁用 + 顶部提示横幅）
- [ ] 只读模式下终端输出仍实时更新

### 验收标准

- 未认证无法访问
- 新设备登录后旧设备自动降级为只读
- 白名单外的设备无法登录

---

## 第七阶段：打磨与交付

**目标：** 产品级体验，单二进制交付

### 构建与部署

- [ ] 前端构建产物通过 `go:embed` 嵌入 Go 二进制
- [ ] Makefile / 构建脚本（前端 build → Go build → 单文件产物）
- [ ] 跨平台编译（Linux、macOS、Windows）

### 体验优化

- [ ] 终端 resize 响应（窗口/容器大小变化时同步 PTY 尺寸）
- [ ] 快捷键支持（新建容器、切换页面等）
- [ ] 页面加载 / 连接中的 loading 状态
- [ ] 错误状态的友好提示
- [ ] 整体 UI 动画与交互细节打磨

### 验收标准

- 单个二进制文件 + config.yaml 即可运行
- 日常使用体验流畅，无明显卡顿或 UI 异常
