# Agent Hive 需求文档 — 文件浏览模式 & Bug 修复

> 创建日期：2026-04-16
> 状态：已确认
> 版本：v1.0

## 1. 项目概述

### 1.1 项目背景
Agent Hive 是基于 Web 的多终端管理器。当前每个项目容器仅有 Todo+终端 一种视图模式，用户无法在 Web 界面中浏览项目文件，需要另开工具查看代码和文档。

### 1.2 项目目标
- 为项目容器新增文件浏览模式，支持只读查看项目目录和文件内容
- 修复终端关闭后未跳转到默认终端的 Bug
- 移除终端数量上限

### 1.3 成功标准
- 用户可在容器标题栏一键切换 Todo+终端 与 文件树+文件预览 模式
- 文件树正确展示终端工作目录内容，支持展开/折叠
- 代码文件语法高亮，Markdown 渲染（含表格、复选框），图片/PDF 可预览
- 桌面端和手机端均可使用，手机端有独立的导航交互

### 1.4 目标用户
使用 Agent Hive 管理 AI Agent 或终端任务的开发者，需要在同一界面中查看项目文件和 Agent 输出。

### 1.5 核心约束
- 只读浏览，不支持文件编辑
- 文件树根路径取自 Agent（默认）终端的当前工作目录，在切换模式时获取
- 需兼容现有桌面端双布局模式（多项目/单项目）和手机端布局

## 2. 功能需求

### 2.1 Bug 修复

#### 2.1.1 关闭额外终端后未跳转到默认终端
- **描述**：关闭所有额外终端后，活动标签未切换到默认终端（Agent），导致无终端处于激活状态
- **原因**：`useTerminalTabs` 的 `handleCloseTerminal` 闭包捕获了过期的 `activeTerminalId`
- **修复方案**：添加 `useEffect` 守卫，当 `activeTerminalId` 指向不存在的终端时自动回退到默认终端
- **验收标准**：
  - Given 用户有 Agent + Terminal 2，当前在 Terminal 2，When 关闭 Terminal 2，Then 自动切换到 Agent 标签
  - Given 用户有 Agent + Terminal 2 + Terminal 3，当前在 Terminal 3，When 依次关闭 Terminal 2 和 Terminal 3，Then 最终切换到 Agent 标签
- **优先级**：P0
- **成熟度**：RS5（已实现）

### 2.2 终端数量上限移除

#### 2.2.1 取消终端数量上限
- **描述**：移除每个容器最多 5 个终端的限制，允许创建任意数量的终端
- **验收标准**：
  - Given 用户已有 5 个终端，When 点击 "+" 按钮，Then 成功创建第 6 个终端
  - Given 任意数量终端，When 查看标签栏，Then "+" 按钮始终可见，标签栏可横向滚动
- **优先级**：P0
- **成熟度**：RS5（已实现）

### 2.3 文件浏览模式

#### 2.3.1 模式切换
- **描述**：每个项目容器可在两种模式间切换：(A) Todo+终端（当前）；(B) 文件树+文件内容预览
- **用户场景**：用户想查看 Agent 正在操作的项目文件，点击标题栏切换按钮进入文件浏览模式
- **输入**：点击容器标题栏上的切换按钮
- **输出**：容器视图切换，文件树加载 Agent 终端当前工作目录
- **验收标准**：
  - Given 容器处于 Todo+终端 模式，When 点击切换按钮，Then 显示文件树+文件内容预览，文件树根目录为 Agent 终端的当前工作目录
  - Given 容器处于文件浏览模式，When 点击切换按钮，Then 返回 Todo+终端 模式，终端状态不丢失
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.2 文件树
- **描述**：展示项目目录结构，支持展开/折叠子目录，显示文件名和目录层级，现代化风格
- **用户场景**：用户在文件树中浏览目录结构，展开感兴趣的目录，点击文件查看内容
- **输入**：点击目录展开/折叠，点击文件选中
- **输出**：目录展开显示子内容，文件选中后在内容区域显示
- **验收标准**：
  - Given 文件树已加载，When 点击一个目录，Then 该目录展开显示子文件和子目录
  - Given 一个目录已展开，When 再次点击，Then 该目录折叠
  - Given 文件树中有隐藏文件（如 `.git`），When 查看文件树，Then 隐藏文件正常显示
  - Given 文件树中有 `node_modules` 等大目录，When 查看文件树，Then 正常显示
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.3 文件内容预览 — 代码文件
- **描述**：代码文件以语法高亮方式只读显示
- **验收标准**：
  - Given 选中一个 `.go` 文件，When 内容加载完成，Then 显示带语法高亮的代码
  - Given 选中一个 `.tsx` 文件，When 内容加载完成，Then 显示带语法高亮的代码
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.4 文件内容预览 — Markdown 渲染
- **描述**：Markdown 文件（包括 README）渲染为富文本显示，支持表格和复选框
- **验收标准**：
  - Given 选中一个包含表格的 `.md` 文件，When 内容加载完成，Then 表格正确渲染为 HTML 表格
  - Given 选中一个包含 `- [x]` 复选框的 `.md` 文件，When 内容加载完成，Then 复选框正确渲染（只读）
  - Given 选中 `README.md`，When 内容加载完成，Then 完整渲染为富文本
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.5 文件内容预览 — 图片
- **描述**：图片文件（PNG, JPG, GIF, SVG 等）在内容区域中预览
- **验收标准**：
  - Given 选中一个 `.png` 文件，When 内容加载完成，Then 图片在内容区域中显示
- **优先级**：P1
- **成熟度**：RS3

#### 2.3.6 文件内容预览 — PDF
- **描述**：PDF 文件在内容区域中预览
- **验收标准**：
  - Given 选中一个 `.pdf` 文件，When 内容加载完成，Then PDF 内容在内容区域中显示
- **优先级**：P1
- **成熟度**：RS3

#### 2.3.7 文件内容预览 — 二进制文件
- **描述**：二进制文件（编译产物、压缩包等）显示"不支持预览"提示
- **验收标准**：
  - Given 选中一个 `.exe` 或 `.zip` 文件，When 内容加载完成，Then 显示"不支持预览"提示信息
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.8 文件内容预览 — 大文件截断
- **描述**：大文件只显示最近的 N 行（默认 1000 行，行数可配置），并提示文件已截断
- **验收标准**：
  - Given 选中一个超过 1000 行的文本文件，When 内容加载完成，Then 显示最近 1000 行，并在顶部提示"文件已截断，显示最后 1000 行"
- **优先级**：P0
- **成熟度**：RS3

#### 2.3.9 桌面端布局
- **描述**：桌面端文件浏览模式采用类 IDE 布局：左侧文件树 + 右侧文件内容
- **验收标准**：
  - Given 桌面端进入文件浏览模式，When 查看布局，Then 左侧为文件树面板，右侧为文件内容面板
  - Given 桌面端文件浏览模式，When 在多项目和单项目布局间切换，Then 文件浏览模式正常显示
- **优先级**：P0
- **成熟度**：RS4

#### 2.3.10 手机端交互
- **描述**：手机端切换到文件浏览模式后显示文件树全屏，点击文件进入文件内容全屏查看，有返回按钮回到文件树
- **验收标准**：
  - Given 手机端进入文件浏览模式，When 查看界面，Then 显示全屏文件树
  - Given 手机端文件树中，When 点击一个文件，Then 进入文件内容全屏视图，顶部有返回按钮
  - Given 手机端文件内容视图，When 点击返回按钮，Then 回到文件树视图
- **优先级**：P0
- **成熟度**：RS4

## 3. 非功能需求

### 3.1 性能要求
- 文件树目录展开响应时间 < 500ms
- 文件内容加载响应时间 < 1s（大文件截断后）
- 大文件截断在后端完成，避免传输过多数据

### 3.2 安全要求
- 文件读取受限于终端工作目录的文件系统权限（使用与 PTY 相同的用户身份）
- 路径遍历防护：禁止 `..` 等路径穿越

### 3.3 兼容性要求
- 与现有桌面端双布局模式兼容
- 与现有手机端布局兼容
- 模式切换不影响终端连接和 Todo 数据

### 3.4 技术约束
- 后端 Go，前端 React + TypeScript
- 语法高亮：**Shiki**（VS Code 同源高亮质量）
- Markdown 渲染：**react-markdown + remark-gfm**（GFM 表格、复选框）+ **@tailwindcss/typography**（排版样式）
- PDF 预览：**react-pdf**（基于 pdf.js 的 React 封装）
- 文件树与文件内容面板分割比例可拖拽调整
- 文件树支持搜索/过滤功能（输入关键字快速定位文件）
