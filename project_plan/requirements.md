# Agent Hive 多终端与布局增强 需求文档

> 创建日期：2026-04-16
> 状态：已审核
> 版本：v1.2

## 1. 项目概述

### 1.1 项目背景
Agent Hive 当前每个项目容器只支持一个终端会话，限制了用户在同一项目下并行操作的能力（如同时运行前端 dev server 和后端服务）。同时桌面端仅有 2×2 网格布局，当用户专注于单个项目时无法充分利用屏幕空间。移动端终端区域偏小，影响操作体验。

### 1.2 项目目标
- 支持每个项目开多个终端，通过标签页切换
- 桌面端提供多项目 / 单项目两种布局模式
- 优化移动端布局，增大终端显示区域
- 移动端快捷键栏补充回车键

### 1.3 成功标准
- 每个项目可创建最多 5 个终端，标签页切换流畅
- 多终端的会话持久化和跨设备同步与现有单终端表现一致
- 桌面端布局可一键切换，单项目模式下终端占满主区域
- 移动端终端区域占比从约 50% 提升到 70%

### 1.4 目标用户
开发者 / AI Agent 使用者，需要在一个项目内并行运行多个终端任务（如前后端服务、日志监控、构建命令等）。

### 1.5 核心约束
- 技术栈不变：Go 后端 + React/TypeScript 前端
- 单二进制交付方式不变
- 仅支持 systemd 系统
- 需兼容现有数据（已有容器平滑升级为支持多终端）

## 2. 功能需求

### 2.1 多终端标签页

#### 2.1.1 终端标签页 UI
- **描述**：每个项目容器内显示终端标签栏，支持通过点击标签页切换不同终端
- **用户场景**：用户在一个项目中需要同时运行前端 dev server 和后端服务，通过标签页快速切换查看
- **输入**：用户点击标签页
- **输出**：切换到对应终端，显示该终端的内容
- **验收标准**：
  - Given 一个项目有 3 个终端，When 用户点击第 2 个标签页，Then 终端区域切换显示第 2 个终端的内容，标签页高亮状态更新
  - Given 一个项目有 1 个终端，When 页面加载，Then 标签栏仍然可见，显示一个标签页
- **优先级**：P0
- **成熟度**：RS3
- **备注**：桌面端和移动端都需要显示标签栏。每个项目有一个固定的默认终端（不可关闭），额外创建的终端命名为 "Terminal 2"、"Terminal 3"……按创建顺序递增

#### 2.1.2 新建终端
- **描述**：标签栏末尾提供"+"按钮，点击新建一个终端标签页
- **用户场景**：用户需要在当前项目中打开一个新的终端
- **输入**：用户点击"+"按钮
- **输出**：新建一个终端标签页并自动切换到该标签页，建立 WebSocket 连接
- **验收标准**：
  - Given 一个项目有 2 个终端，When 用户点击"+"，Then 新建第 3 个终端并自动切换过去，标签栏显示 3 个标签
  - Given 一个项目已有 5 个终端，When 用户查看标签栏，Then "+"按钮不显示或禁用
- **优先级**：P0
- **成熟度**：RS3
- **备注**：每个项目最多 5 个终端

#### 2.1.3 关闭终端
- **描述**：额外创建的终端标签页上提供"x"关闭按钮，点击关闭该终端
- **用户场景**：用户完成某项任务，不再需要该终端
- **输入**：用户点击标签页上的"x"按钮
- **输出**：终端关闭，PTY 会话销毁，标签页移除，自动切换到相邻标签页
- **验收标准**：
  - Given 终端内没有运行中的前台进程，When 用户点击"x"，Then 终端直接关闭，不弹确认
  - Given 终端内有运行中的前台进程，When 用户点击"x"，Then 弹出确认对话框，用户确认后关闭
  - Given 一个项目只剩默认终端，When 用户查看该标签页，Then "x"关闭按钮不显示（默认终端不可关闭）
- **优先级**：P0
- **成熟度**：RS3
- **备注**：关闭后自动切换到相邻标签页（优先右侧，右侧没有则左侧）

#### 2.1.4 多终端持久化
- **描述**：每个终端的会话独立持久化，服务重启后所有终端标签页及其历史内容恢复
- **用户场景**：服务重启后，用户打开浏览器看到之前的所有终端标签页和输出历史
- **输入**：服务重启
- **输出**：所有项目的所有终端标签页恢复，终端输出历史保留
- **验收标准**：
  - Given 一个项目有 3 个终端且各有输出历史，When 服务重启后用户访问页面，Then 3 个标签页都在，切换每个标签页都能看到之前的输出历史
  - Given 旧版数据库中有容器（单终端模式），When 升级后首次启动，Then 每个容器自动拥有一个终端标签页，原有历史数据保留
- **优先级**：P0
- **成熟度**：RS3
- **备注**：默认终端恢复历史上限为 1000 行，额外创建的终端恢复历史上限为 200 行，超出部分从头部截断

#### 2.1.5 多终端跨设备同步
- **描述**：终端的创建、关闭、输入输出在所有已连接设备间实时同步
- **用户场景**：用户在电脑上新建了一个终端，手机上立即看到新标签页
- **输入**：任意设备上的终端操作
- **输出**：其他设备实时同步显示
- **验收标准**：
  - Given 设备 A 和设备 B 都连接到同一项目，When 设备 A 新建一个终端，Then 设备 B 的标签栏实时出现新标签页
  - Given 设备 A 在终端 2 输入命令，When 命令产生输出，Then 设备 B 切换到终端 2 也能看到输出
  - Given 设备 A 和设备 B 都连接到同一项目，When 设备 A 关闭一个终端标签页，Then 设备 B 的标签栏实时移除该标签页
- **优先级**：P0
- **成熟度**：RS3

#### 2.1.6 终端懒加载
- **描述**：默认终端始终保持 WebSocket 连接和渲染状态（切换到其他标签页时不销毁）；额外创建的终端采用懒加载，仅在切换到对应标签页时才建立连接和加载内容，切换离开后释放
- **用户场景**：用户频繁在默认终端和其他终端间切换，默认终端需要即时响应不能有加载延迟；额外终端主要用于临时调试，不需要常驻
- **输入**：用户切换终端标签页
- **输出**：默认终端始终可用；额外终端按需加载
- **验收标准**：
  - Given 一个项目有 3 个终端，When 用户切换到标签 2（额外终端），Then 默认终端仍保持连接，标签 2 建立连接并加载内容
  - Given 用户从标签 2 切换回默认终端，When 切换完成，Then 默认终端立即显示最新内容（无加载延迟），标签 2 释放连接
  - Given 一个项目有 3 个终端，When 页面加载，Then 默认终端自动建立连接，其余 2 个不建立连接
  - Given 设备 A 和设备 B 都打开了同一个终端标签页，When 设备 A 在该终端输入命令，Then 设备 B 实时看到输出（已激活的同一标签页跨设备同步）
  - Given 设备 A 打开了标签 1，设备 B 打开了标签 2，When 设备 A 在标签 1 输入命令，Then 设备 B 不受影响（未激活标签不加载）
- **优先级**：P0
- **成熟度**：RS3

### 2.2 桌面端布局切换

#### 2.2.1 布局切换按钮
- **描述**：桌面端页面顶部添加布局切换按钮，在多项目模式（2×2 网格）和单项目模式（全屏）之间切换
- **用户场景**：用户想专注于一个项目的操作，切换到单项目模式获得更大的终端区域
- **输入**：用户点击布局切换按钮
- **输出**：页面布局在多项目和单项目之间切换
- **验收标准**：
  - Given 当前为多项目模式，When 用户点击布局切换按钮，Then 页面切换为单项目模式，当前聚焦的项目（最近一次获得终端焦点的项目）占满页面
  - Given 当前为单项目模式，When 用户点击布局切换按钮，Then 页面恢复为 2×2 网格
  - Given 用户切换到单项目模式后刷新页面，When 页面加载，Then 仍为单项目模式（布局状态已持久化）
- **优先级**：P0
- **成熟度**：RS3
- **备注**：布局状态需要持久化，刷新后保持当前布局模式

#### 2.2.2 单项目模式
- **描述**：单项目模式下，一个项目占满页面主区域，保留待办列表（可拖拽调整比例），终端通过标签页切换
- **用户场景**：用户专注于一个项目，需要更大的终端和待办列表空间
- **输入**：切换到单项目模式
- **输出**：当前项目全屏显示，左侧待办列表 + 右侧终端标签页和终端
- **验收标准**：
  - Given 单项目模式，When 页面渲染，Then 项目占满页面主区域，待办列表可见，终端标签页可切换
  - Given 单项目模式显示项目 A，When 用户点击标题栏右箭头或按 Ctrl+→，Then 切换到下一个项目
  - Given 单项目模式显示第一个项目，When 用户点击标题栏左箭头或按 Ctrl+←，Then 无响应（已是第一个）
  - Given 单项目模式显示最后一个项目，When 用户点击标题栏右箭头或按 Ctrl+→，Then 无响应（已是最后一个）
- **优先级**：P0
- **成熟度**：RS3
- **备注**：项目切换方式：标题栏左右箭头按钮 + Ctrl+←/→ 快捷键，与移动端滑动切换体验对齐

### 2.3 移动端布局调整

#### 2.3.1 移动端 7/3 布局
- **描述**：移动端布局默认 7/3 分——上方 70% 显示终端标签页、终端内容和快捷键栏，下方 30% 显示项目名称和待办列表。支持拖动分割线调整比例
- **用户场景**：用户在手机上操作终端，需要更大的终端显示区域，终端在上方便于操作；有时需要查看更多待办内容，可拖动分割线调整
- **输入**：移动端页面加载 / 用户拖动分割线
- **输出**：页面按 7/3 默认比例分割，终端在上，待办在下；拖动分割线可调整比例
- **验收标准**：
  - Given 移动端访问，When 页面加载，Then 终端区域（含标签栏和快捷键栏）约占屏幕上方 70%，项目信息和待办约占下方 30%
  - Given 移动端页面已加载，When 用户拖动分割线向下，Then 终端区域增大，待办区域缩小
  - Given 移动端页面已加载，When 用户拖动分割线向上，Then 待办区域增大，终端区域缩小
  - Given 移动端页面已加载，When 用户拖动分割线到极限位置，Then 终端区域最小 40%，待办区域最小 15%
- **优先级**：P1
- **成熟度**：RS3

### 2.4 移动端快捷键栏增强

#### 2.4.1 新增回车键
- **描述**：移动端快捷键栏新增回车键（Enter）按钮
- **用户场景**：用户在移动端终端需要快速按回车确认命令
- **输入**：用户点击回车键按钮
- **输出**：向终端发送回车字符（\r）
- **验收标准**：
  - Given 移动端快捷键栏可见，When 用户点击回车键按钮，Then 终端收到回车输入
- **优先级**：P1
- **成熟度**：RS3

## 3. 非功能需求

### 3.1 性能要求
- 标签页切换响应时间 < 100ms（用户感知即时）
- 多终端模式下（最多 20 个 WebSocket 连接）不影响现有终端交互流畅度
- 暂不做特别优化，后续按需调整

### 3.2 安全要求
- 与现有安全模型一致（可选密码认证）
- 无新增安全需求

### 3.3 兼容性要求
- 桌面端：主流浏览器（Chrome、Firefox、Edge、Safari）
- 移动端：iOS Safari、Android Chrome
- 与现有兼容范围一致

### 3.4 技术约束
- 技术栈不变：Go + React/TypeScript + xterm.js + SQLite
- 单二进制交付
- 仅支持 systemd（Linux）
- 需兼容现有数据结构，已有容器平滑升级支持多终端

## 4. 术语表

| 术语 | 定义 |
|------|------|
| 项目容器 | Agent Hive 中的一个工作单元，包含名称、待办列表和一组终端 |
| 终端标签页 | 项目容器内的一个 PTY 终端会话，通过标签页 UI 切换 |
| 多项目模式 | 桌面端 2×2 网格布局，一页显示 4 个项目容器 |
| 单项目模式 | 桌面端全屏布局，一个项目容器占满页面 |
| 默认终端 | 每个项目容器创建时自动生成的第一个终端，不可关闭，始终保持连接和渲染状态 |
| 当前聚焦项目 | 多项目模式下，最近一次获得终端焦点（点击/输入）的项目容器 |
