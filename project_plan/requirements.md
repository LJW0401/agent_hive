# Agent Hive 移动端适配 需求文档

> 创建日期：2026-04-14
> 状态：已审核
> 版本：v1.1

## 1. 项目概述

### 1.1 项目背景
Agent Hive 当前的 2×2 网格布局面向桌面端设计，在手机小屏幕上无法正常使用。需要针对移动端的屏幕特点提供专门的布局和交互方式，使用户能在手机上高效管理终端会话和待办事项。

### 1.2 项目目标
- 实现手机端单项目全屏视图，提供良好的移动端操作体验
- 支持手势滑动切换项目、拖拽调整终端与待办区域比例
- 手机端布局独立于桌面端存储和恢复

### 1.3 成功标准
- 手机浏览器访问时自动进入移动端布局，无需手动切换
- 终端在手机端可输入命令、查看输出；待办事项可增删改查及拖拽排序
- 项目排序在关闭浏览器后重新打开可恢复

### 1.4 目标用户
已在使用 Agent Hive 的用户，希望通过手机随时查看和操作终端会话与待办事项。

### 1.5 核心约束
- 前端技术栈沿用 React + TypeScript + Tailwind CSS
- 必须与现有桌面端功能兼容，不影响桌面端体验
- 单二进制交付方式不变

## 2. 功能需求

### 2.1 设备检测

#### 2.1.1 User-Agent 设备识别
- **描述**：通过浏览器 User-Agent 字符串判断当前设备为手机或电脑，自动选择对应的布局模式
- **用户场景**：用户在手机浏览器中打开 Agent Hive 页面
- **输入**：浏览器 User-Agent 字符串
- **输出**：设备类型标识（mobile / desktop）
- **验收标准**：
  - Given 用户使用 iPhone Safari 访问, When 页面加载完成, Then 自动进入移动端单项目视图
  - Given 用户使用桌面 Chrome 访问, When 页面加载完成, Then 显示原有 2×2 网格布局
  - Given 用户使用平板设备（iPad 等）访问, When 页面加载完成, Then 显示桌面端 2×2 网格布局
- **优先级**：P0
- **成熟度**：RS3

### 2.2 移动端布局

#### 2.2.1 单项目全屏视图
- **描述**：手机端一页只显示一个项目，上方为终端区域，下方为待办事项区域
- **用户场景**：用户在手机上查看某个项目的终端输出和待办事项
- **输入**：当前选中的项目
- **输出**：全屏展示该项目的终端（上）和待办事项（下）
- **验收标准**：
  - Given 用户在手机端打开页面, When 页面加载完成, Then 显示第一个项目，终端在上、待办在下，占满屏幕
  - Given 用户在手机端查看项目, When 没有任何项目存在, Then 显示新建项目页面
- **优先级**：P0
- **成熟度**：RS3

#### 2.2.2 终端与待办区域可拖拽分割
- **描述**：终端区域和待办区域之间有一条分割线，用户可以上下拖拽该分割线来自由调整两个区域的高度比例
- **用户场景**：用户希望在手机上更多地查看终端输出，拖拽分割线将终端区域扩大
- **输入**：用户在分割线上进行上下拖拽手势
- **输出**：终端和待办区域高度实时跟随手指变化
- **验收标准**：
  - Given 用户在手机端查看项目, When 在分割线上向上拖拽, Then 待办区域扩大、终端区域缩小
  - Given 用户在手机端查看项目, When 在分割线上向下拖拽, Then 终端区域扩大、待办区域缩小
  - Given 用户拖拽分割线, When 拖拽至极限位置, Then 不会完全隐藏任一区域（最小高度为各自一行内容的高度）
- **优先级**：P0
- **成熟度**：RS3

### 2.3 项目切换

#### 2.3.1 左右滑动切换项目
- **描述**：用户在手机端通过左右滑动手势在项目之间切换。通过触摸滑动的总体方向判断意图——以 45° 为分界，偏水平方向触发项目切换，偏垂直方向触发终端内滚动，避免手势冲突
- **用户场景**：用户想查看下一个项目的终端状态
- **输入**：左滑或右滑手势（水平方向为主）
- **输出**：切换到相邻的项目视图
- **验收标准**：
  - Given 用户在第一个项目页面, When 向左滑动, Then 切换到第二个项目
  - Given 用户在最后一个项目页面, When 向左滑动, Then 切换到新建项目页面
  - Given 用户在第一个项目页面, When 向右滑动, Then 无反应或弹性回弹
  - Given 用户在终端区域, When 以偏垂直方向滑动, Then 触发终端内滚动而非项目切换
- **优先级**：P0
- **成熟度**：RS3

### 2.4 项目排序

#### 2.4.1 左右按键移动项目位置
- **描述**：项目页面上提供左右箭头按钮，点击后将当前项目在排序中前移或后移一位
- **用户场景**：用户想把常用项目调整到前面方便快速访问
- **输入**：点击左箭头或右箭头按钮
- **输出**：当前项目在排列中前移或后移一位，视图跟随移动
- **验收标准**：
  - Given 用户在第二个项目页面, When 点击左箭头, Then 该项目移到第一位，视图仍显示该项目
  - Given 用户在第一个项目页面, When 点击左箭头, Then 按钮禁用或无反应
  - Given 用户在最后一个项目页面, When 点击右箭头, Then 按钮禁用或无反应
- **优先级**：P0
- **成熟度**：RS3

### 2.5 项目管理操作

#### 2.5.1 移动端项目删除与重命名
- **描述**：移动端项目标题栏提供与桌面端一致的操作按钮，包括重命名和删除（关闭）按钮
- **用户场景**：用户在手机上想修改项目名称或关闭不再需要的项目
- **输入**：点击标题栏上的重命名或删除按钮
- **输出**：重命名时弹出编辑框；删除时关闭项目并自动切换到相邻项目
- **验收标准**：
  - Given 用户在移动端查看项目, When 点击重命名按钮并输入新名称, Then 项目名称更新，多设备同步
  - Given 用户在移动端查看项目, When 点击删除按钮, Then 项目被关闭，视图自动切换到前一个项目
  - Given 用户删除唯一的项目, When 删除完成, Then 显示新建项目页面
- **优先级**：P0
- **成熟度**：RS3

### 2.6 新建项目入口

#### 2.6.1 末尾新建项目页面
- **描述**：在所有项目的最后一页放置一个"新建项目"页面，用户滑动到末尾即可创建新项目
- **用户场景**：用户想在手机上创建一个新的终端会话
- **输入**：用户滑动到最后一页，点击新建按钮
- **输出**：创建新项目并跳转到该项目的视图
- **验收标准**：
  - Given 用户滑动到最后一页, When 页面显示, Then 展示新建项目入口（与桌面端风格一致的虚线框按钮）
  - Given 用户点击新建按钮, When 项目创建成功, Then 新项目插入到新建页面之前，视图自动切换到新项目
  - Given 用户点击新建按钮, When 创建失败（如网络异常）, Then 显示错误提示，保持在新建页面
- **优先级**：P0
- **成熟度**：RS3

### 2.7 布局持久化

#### 2.7.1 手机端布局独立存储
- **描述**：手机端的项目排列顺序独立于桌面端存储在后端数据库中，所有手机共享同一套移动端布局，两套布局（移动端/桌面端）互不影响
- **用户场景**：用户在手机上调整了项目顺序，不希望影响电脑上的 2×2 网格排列
- **输入**：手机端项目排序变更
- **输出**：后端保存手机端专用的布局数据
- **验收标准**：
  - Given 用户在手机端调整了项目顺序, When 在电脑端访问, Then 电脑端布局不受影响
  - Given 用户在手机端调整了项目顺序后关闭浏览器, When 重新用手机打开, Then 恢复之前的排列顺序
  - Given 新创建的项目, When 在手机端和桌面端分别查看, Then 均出现在各自布局中
- **优先级**：P0
- **成熟度**：RS3

## 3. 非功能需求

### 3.1 性能要求
- 手机端页面首次加载时间 < 3s（局域网环境）
- 左右滑动切换项目动画流畅，无明显卡顿（目标 60fps）

### 3.2 安全要求
- 沿用现有密码认证机制，无额外安全需求

### 3.3 兼容性要求
- Safari (iOS)
- Chrome (Android)
- Firefox (移动端)
- Edge (移动端)

### 3.4 技术约束
- 前端：React + TypeScript + Tailwind CSS + xterm.js
- 后端：Go + SQLite
- 交付方式：go:embed 单二进制文件
